## Repository Purpose

IICPC competitive programming challenge workspace. Solutions are self-contained programs; no build system or test runner is configured at the repo level.

## Structure

Each problem will likely live in its own file or subdirectory. `PROTOCOL.md` documents submission guidelines (currently a stub).

## Conventions

- Language choice is per-problem — check existing files in a problem directory before assuming a language.
- Solutions are standalone; no shared libraries or imports across problems unless explicitly set up.
- Follow submission format specified in `PROTOCOL.md` once populated.
